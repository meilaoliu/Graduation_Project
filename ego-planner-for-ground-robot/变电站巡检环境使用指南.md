# 变电站巡检环境使用指南

## 环境概述

本变电站巡检环境是基于 Gazebo 仿真器创建的专业巡检测试场景，专门为地面巡检机器人的路径规划和自主导航研究设计。

### 环境特点

- **真实场景模拟**：模拟真实变电站的设备布局和空间结构
- **丰富的障碍物**：包含变压器、配电柜、开关设备等典型电力设备
- **安全边界**：完整的围栏系统模拟真实安全防护
- **巡检路径**：预设巡检点和巡检路径
- **可扩展性**：支持添加更多设备和传感器

### 设备清单

| 设备类型 | 数量 | 尺寸(m) | 颜色 | 位置说明 |
|---------|------|---------|------|----------|
| 主变压器 | 2 | 4×3×3 | 灰色 | 场地东侧 |
| 配电柜 | 3 | 2×1×2.5 | 蓝色 | 场地西侧 |
| 开关设备 | 2 | 1.5×0.8×2 | 黄色 | 场地南北两侧 |
| 高压电塔 | 2 | 1×1×8 | 灰色 | 场地东侧边缘 |
| 控制室 | 1 | 8×6×3 | 白色 | 场地西南角 |
| 巡检标志点 | 3 | Φ0.6×0.1 | 红色 | 关键设备附近 |
| 围栏 | - | 50×50×2 | 绿色 | 场地外围 |

## 快速启动

### 1. 启动变电站仿真环境

```bash
cd ~/Graduation_Project/ego-planner-for-ground-robot
source devel/setup.bash
roslaunch vehicle_simulator system_substation.launch
```

### 2. 启动路径规划器

在新终端中运行：

```bash
cd ~/Graduation_Project/ego-planner-for-ground-robot
source devel/setup.bash
roslaunch plan_manage run_in_sim.launch
```

### 3. 设置巡检目标点

在 RViz 中使用 **2D Nav Goal** 工具设置目标点，建议巡检路径：

1. 入口点：(-20, 0)
2. 配电柜区域：(-8, 0)
3. 变压器1：(10, 0)
4. 变压器2：(10, 15)
5. 开关设备1：(5, -10)
6. 开关设备2：(5, 10)
7. 返回入口

## 详细配置

### 启动参数说明

```bash
roslaunch vehicle_simulator system_substation.launch \
    vehicleX:=-20 \           # 机器人初始X坐标
    vehicleY:=0 \             # 机器人初始Y坐标
    vehicleYaw:=0 \           # 机器人初始朝向(弧度)
    vehicleHeight:=0.75 \     # 机器人高度
    gazebo_gui:=true \        # 是否显示Gazebo界面
    checkTerrainConn:=false   # 是否检查地形连通性
```

## 3D模型制作指南

### 当前模型限制

目前环境中的设备都是简单的几何体（长方体、圆柱体），缺乏细节和真实感。为了提升仿真真实度，可以制作更精细的3D模型。

### 模型制作流程

#### 1. 3D建模软件选择

**推荐软件：**
- **Blender**（免费，功能强大）
- **SketchUp**（简单易用）
- **SolidWorks**（专业建模）
- **Fusion 360**（免费学生版）

#### 2. 变电站设备建模要点

**变压器建模：**
```bash
# 主要特征
- 主体：圆柱形或方形外壳
- 散热器：波纹状散热片
- 绝缘子：瓷质绝缘组件
- 套管：高压进出线套管
- 冷却装置：散热风扇或油冷装置
```

**配电柜建模：**
```bash
# 主要特征  
- 柜体：标准机柜结构
- 门板：带观察窗
- 内部设备：开关、仪表、线缆
- 指示灯：运行状态指示
- 铭牌：设备标识
```

#### 3. 模型文件格式

Gazebo支持的模型格式：
- **.dae**（推荐，COLLADA格式）
- **.obj**（Wavefront OBJ）
- **.stl**（STL格式）
- **.mesh**（Ogre格式）

#### 4. 模型制作标准

```bash
# 几何要求
- 面数控制：单个模型 < 5000面
- 尺寸准确：按真实设备比例
- 坐标原点：模型底部中心
- 单位：米(m)

# 材质要求
- 纹理分辨率：512×512 或 1024×1024
- 格式：PNG、JPG
- UV展开：合理的UV映射
- 材质简洁：避免过于复杂的材质
```

### 模型集成步骤

#### 1. 创建模型目录结构

```bash
mkdir -p ~/.gazebo/models/transformer_realistic
cd ~/.gazebo/models/transformer_realistic

# 创建标准目录结构
mkdir meshes materials textures
```

#### 2. 创建模型配置文件

```xml
<!-- model.config -->
<?xml version="1.0"?>
<model>
  <name>transformer_realistic</name>
  <version>1.0</version>
  <sdf version="1.6">model.sdf</sdf>
  
  <author>
    <name>Your Name</name>
    <email>your.email@example.com</email>
  </author>
  
  <description>
    Realistic transformer model for substation simulation
  </description>
</model>
```

#### 3. 创建SDF模型文件

```xml
<!-- model.sdf -->
<?xml version="1.0" ?>
<sdf version="1.6">
  <model name="transformer_realistic">
    <static>true</static>
    <link name="link">
      <collision name="collision">
        <geometry>
          <mesh>
            <uri>model://transformer_realistic/meshes/transformer.dae</uri>
            <scale>1 1 1</scale>
          </mesh>
        </geometry>
      </collision>
      
      <visual name="visual">
        <geometry>
          <mesh>
            <uri>model://transformer_realistic/meshes/transformer.dae</uri>
            <scale>1 1 1</scale>
          </mesh>
        </geometry>
        <material>
          <script>
            <uri>model://transformer_realistic/materials/scripts/transformer.material</uri>
            <name>transformer/diffuse</name>
          </script>
        </material>
      </visual>
    </link>
  </model>
</sdf>
```

#### 4. 在world文件中使用新模型

```xml
<!-- 在 substation.world 中替换原有模型 -->
<include>
  <uri>model://transformer_realistic</uri>
  <pose>10 0 1.5 0 0 0</pose>
  <name>transformer_1</name>
</include>
```

### 材质和纹理制作

#### 1. 材质脚本示例

```bash
# materials/scripts/transformer.material
material transformer/diffuse
{
    technique
    {
        pass
        {
            texture_unit
            {
                texture transformer_diffuse.png
                tex_coord_set 0
            }
        }
    }
}
```

#### 2. 纹理制作要点

```bash
# 变压器纹理
- 金属外壳：灰色金属质感
- 散热器：铝制波纹表面
- 绝缘子：白色瓷质材料
- 标识牌：清晰的文字标识

# 配电柜纹理
- 柜体：深蓝或灰色喷塑
- 指示灯：红绿黄LED指示
- 仪表盘：白底黑字仪表
- 线缆：不同颜色的电缆
```

### 高级功能集成

#### 1. 添加传感器模拟

```xml
<!-- 红外热成像传感器 -->
<sensor name="thermal_camera" type="camera">
  <pose>0 0 2 0 0 0</pose>
  <camera>
    <horizontal_fov>1.047</horizontal_fov>
    <image>
      <width>640</width>
      <height>480</height>
    </image>
  </camera>
  <plugin name="thermal_camera_plugin" filename="libgazebo_ros_camera.so">
    <robotNamespace>/</robotNamespace>
    <cameraName>thermal_camera</cameraName>
    <imageTopicName>thermal_image</imageTopicName>
  </plugin>
</sensor>
```

#### 2. 动态元素添加

```xml
<!-- 可旋转的设备指示器 -->
<joint name="indicator_joint" type="revolute">
  <pose>0 0 0 0 0 0</pose>
  <parent>link</parent>
  <child>indicator_link</child>
  <axis>
    <xyz>0 0 1</xyz>
    <limit>
      <lower>-3.14159</lower>
      <upper>3.14159</upper>
    </limit>
  </axis>
</joint>
```


### 2. 仿真性能优化

```bash
# 模型优化
- 减少面数：使用Decimate修改器
- 简化材质：减少纹理数量
- 合并对象：减少绘制调用
- LOD系统：距离层次细节

# Gazebo优化
- 实时因子：<real_time_factor>0.5</real_time_factor>
- 更新频率：<real_time_update_rate>500</real_time_update_rate>
- 物理精度：<max_step_size>0.002</max_step_size>
```
